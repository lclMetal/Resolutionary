1. Getting started

Instructions on how to get Resolutionary, and how to merge it into your game.

  1. Download the most up-to-date version of Resolutionary as a .zip package via the link above.
  2. Use a program like WinRAR or 7-Zip to extract the files from the .zip.
  3. Open your game in Game Editor.
  4. Select File -> Merge and navigate to the folder where you extracted the files from the .zip. Select "Resolutionary_template.ged" as the file to be merged into your game. It can be found inside the folder "Template".

2. Initial configuration

These are the steps that you have to take after merging Resolutionary to your game for deploying the system.

  1. (Optional) Use resSetBackgroundColor(); to set a background color for your game. If you don't specify a color, the color black will be used by default.
  2. (Optional) Use resSetBorderColor(); to set a color for the borders that will appear if the game is scaled to a resolution with an aspect ratio differing from the original resolution. These borders are often called "black bars", but you can use any color you like. If you don't specify a color, the color black will be used by default.
  3. Use resStart(); to start the Resolutionary system.
  4. Use resQuit(); in a Key Down event of your choice (excluding Esc) to quit the Resolutionary system before exiting the Game Editor game mode. An exported game doesn't need this event, but for testing your game in Game Editor this is important for avoiding memory leaks. More information can be found on the Resolutionary Reference page.
  5. Resize the Filled Region actor called resActorDetector by dragging its corner so that it is the same size as the view actor in the original resolution of the game.

The calls to resSetBackgroundColor();, resSetBorderColor(); and resStart(); can, for example, be placed in the Create Actor event of view, so that the system will start up at the same moment as the game itself.

// view -> Create Actor

resSetBackgroundColor(128, 128, 128); // Make the background gray
resSetBorderColor(0, 0, 0);           // Make the borders black
resStart(640, 480);                   // Start Resolutionary (the original resolution of the game is 640 x 480)

3. Changing the resolution

These are the steps that you have to take every time you want to change the resolution of your game.

  1. Set the resolution of your choice for the game from Config -> Game Properties -> Game resolution.
  2. Resize the actors resDynamicLayerCanvas and resStaticLayerCanvas so that they are the same size as the view actor.

4. Making your project Resolutionary compatible

In order to have your game function correctly after changing its resolution with Resolutionary, you'll have to make some changes in it.

  • Actions in Out of Vision events have to be moved to Collision Finish from resActorDetector.
  • Actions in Mouse Enter events have to be moved to Collision with resMouse.
  • Actions in Mouse Leave events have to be moved to Collision Finish from resMouse.
  • Actions in Mouse Button Down events have to be moved to Activation Event from resMouseClickDetector.
    • For further info, see functions resOnMouseClick();, resOnMouseButtonDown(); and resIsMouseButtonDown();.
  • Actions in Mouse Button Up events have to be moved to Activation Event from resMouseClickDetector.
    • For further info, see functions resOnMouseRelease();, resOnMouseButtonUp();, and resIsMouseButtonUp();.

2. Initial configuration

2.1 Set background color

Use resSetBackgroundColor(); to set the background color for your game. If you don't specify a background color, the color black is used by default.

A good place for calling resSetBackgroundColor(); is in the Create Actor event of the view actor.

2.2 Set border color

Use resSetBorderColor(); to set the color for the borders that will appear if the game is scaled to a resolution with an aspect ratio differing from the original resolution. These borders are often called "black bars", but you can use any color you like. If you don't specify a color, the color black is used by default.

A good place for calling resSetBorderColor(); is in the Create Actor event of the view actor.

2.3 Set system start event

Use resStart(); to start the Resolutionary system.

A good place for calling resStart(); is in the Create Actor event of the view actor.

2.4 Set system quit event

Use resQuit(); in a Key Down event to quit the Resolutionary system before exiting the Game Editor game mode. An exported game doesn't need this event, but for testing your game in Game Editor this is important for avoiding memory leaks. More information can be found on the Resolutionary Reference page under resQuit();.

For convenience, you can use the Key Up event for the same key for calling ExitGame(); in order to return to Game Editor's edit view from game mode. This way, pressing the key will quit Resolutionary, and releasing it will bring you back to the edit view.

2.5 Resize resActorDetector

Resize the Filled Region actor called resActorDetector by dragging its corner so that it is the same size as the view actor in the original resolution of the game.
