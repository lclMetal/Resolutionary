User functions

void resChangeActorType(const char *actorName, int type)

This function sets an actor's type that specifies how Resolutionary should handle the actor. Dynamic actors will be redrawn every frame, while Static actors will be redrawn only when requested or when the list of static actors changes (new actor is added or old deleted). Not drawable actors will be ignored by Resolutionary's scaling functionality.

Parameters

  • actorName - The name or clonename of any actor in the game. Also supports "Event Actor", "Parent Actor", "Creator Actor" and "Collide Actor".
  • type - The type to be set for the actor, must be one of the following:
    • RES_ACTOR_DYNAMIC
    • RES_ACTOR_STATIC
    • RES_ACTOR_NOT_DRAWABLE

Example call

resChangeActorType("Event Actor", RES_ACTOR_DYNAMIC);

void resChangeActorZLayer(const char *actorName, int zLayer)

This function sets an actor's z layer. Actors are drawn in order by their z layer, from lowest to highest. So, actors with a higher z layer will be drawn on top of the actors with a lower z layer. The number of an actor's z layer can be positive or negative.

Parameters

  • actorName - The name or clonename of any actor in the game. Also supports "Event Actor", "Parent Actor", "Creator Actor" and "Collide Actor".
  • zLayer - The number of the z layer to assign to the actor.

Example call

resChangeActorZLayer("Event Actor", 1);

void resChangeActorParent(const char *actorName, const char *parentName)

This function lets Resolutionary know that an actor has a parent, make sure to use this function together with Game Editor's ChangeParent function if the parented actor is to be drawn by Resolutionary.

Important
Make sure to always use this function together with Game Editor's ChangeParent function if the parented actor is to be drawn by Resolutionary. If the system isn't notified of the change in the actor's parenting, the actor will be drawn in a wrong position.

Parameters

  • actorName - The name or clonename of the child actor. Supports "Event Actor", "Parent Actor", "Creator Actor" and "Collide Actor".
  • parentName - The name or clonename of the parent actor. Supports "Event Actor", "Parent Actor", "Creator Actor" and "Collide Actor".

Example call

resChangeActorParent("Event Actor", "view");
